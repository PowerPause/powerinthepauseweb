# AI Discoverability & Generative Engine Optimization (GEO) - Implementation Brief

**Version:** 1.0
**Purpose:** A reusable brief for making any website visible and citable by AI systems (ChatGPT, Claude, Perplexity, Gemini, Google AI Overviews) alongside traditional search engines. Apply this to any web project.

**Why this matters:** AI-referred visitors convert at 18% vs 2-5% from traditional search. 58% of users have already replaced traditional search with AI tools for product and service discovery. 69% of Google searches now end without a click - the answer is generated by AI instead. The window for early-mover advantage in GEO is 6-12 months.

## PART 2: ROBOTS.TXT

Create or update the robots.txt file at the site root. This controls which bots can crawl the site.

```
# Allow all legitimate crawlers
User-agent: *
Allow: /

# Search engine crawlers (explicit)
User-agent: Googlebot
Allow: /

User-agent: Bingbot
Allow: /

User-agent: Slurp
Allow: /

User-agent: DuckDuckBot
Allow: /

# AI crawlers - these power ChatGPT, Claude, Perplexity, Gemini etc.
# IMPORTANT: Cloudflare recently changed defaults to BLOCK AI bots.
# Verify these are not being overridden by your CDN/WAF settings.
User-agent: ClaudeBot
Allow: /

User-agent: ChatGPT-User
Allow: /

User-agent: GPTBot
Allow: /

User-agent: PerplexityBot
Allow: /

User-agent: Google-Extended
Allow: /

User-agent: Applebot-Extended
Allow: /

User-agent: cohere-ai
Allow: /

# Block known SEO scrapers (optional - adjust to your preference)
User-agent: AhrefsBot
Disallow: /

User-agent: SemrushBot
Disallow: /

User-agent: MJ12bot
Disallow: /

User-agent: DotBot
Disallow: /

# Reference files
Sitemap: https://example.com/sitemap.xml
llms.txt: https://example.com/llms.txt
```

### Framework-Specific Notes

**Next.js:** Place in `public/robots.txt` or use `next-sitemap` package for programmatic generation.
**Nuxt:** Place in `static/robots.txt` or use `@nuxtjs/robots` module.
**Gatsby:** Place in `static/robots.txt`.
**Plain HTML / Static:** Place in site root.
**WordPress:** Use Yoast SEO or similar plugin, or place file manually in webroot.

### Cloudflare Warning

Cloudflare changed its default bot management settings in late 2025 to block AI crawlers automatically. Even if your robots.txt allows them, Cloudflare may be returning 403s before the crawler ever reaches your server.

**Check and fix in Cloudflare dashboard:**

1. Security > Bots > Bot Fight Mode - set to allow Verified Bots
2. Security > Bots > check "AI Crawlers" section - ensure NOT set to Block
3. Security > Settings > Security Level - should be "Medium", NOT "I'm Under Attack"
4. Security > Settings > Browser Integrity Check - consider disabling if legitimate bots are being blocked
5. Rules > Custom Rules - check for rules blocking by user-agent

**Verify after changes:**
```bash
curl -s -A "ClaudeBot/1.0" https://example.com/ | head -20
curl -s -A "GPTBot/1.0" https://example.com/ | head -20
# Both should return HTML content, not a challenge page
```

## PART 3: LLMS.TXT

### What It Is

llms.txt is a plain-text markdown file at the site root that provides AI crawlers with a structured, curated summary of the site. Think of it as sitemap.xml for AI systems. Proposed by Jeremy Howard (fast.ai) in September 2024. Adopted by Anthropic, Cloudflare, Stripe, and 844,000+ websites.

### Template

Create at `https://example.com/llms.txt`:

```markdown
# [Company / Project Name]

> [One-paragraph description of what this company/project does, who it serves, and what makes it distinctive. Write factually - AI systems process content literally. No marketing language.]

## [Section 1 - e.g. Company / About]

- [Page Title](https://example.com/page-url/): [One-sentence description of what this page contains and why it matters.]
- [Page Title](https://example.com/another-page/): [Description.]

## [Section 2 - e.g. Products / Services]

- [Page Title](https://example.com/products/): [Description.]
- [Page Title](https://example.com/services/): [Description.]

## [Section 3 - e.g. Resources / Blog / Documentation]

- [Page Title](https://example.com/blog/): [Description.]

## Key Facts

- Headquarters: [Location]
- Founded: [Year]
- Industry: [Industry]
- Contact: [Email]
- [Any other key facts an AI system should know]
```

### Guidelines

- Keep under 4,000 tokens (roughly 3,000 words) to fit within LLM context windows
- Write factually and precisely - no superlatives, no marketing language
- Include only public, indexable pages
- Update whenever pages are added, removed, or significantly changed
- Do not include internal/staging URLs or authentication-protected pages
- Set a CDN cache TTL of 1 hour and purge on updates

### HTML Integration

Add this `<link>` tag to the `<head>` of every page:

```html
<link rel="llms" href="https://example.com/llms.txt" />
```

## PART 4: MARKDOWN PAGE VERSIONS

### What This Is

AI crawlers extract content from HTML but complex layouts, JavaScript, navigation, cookie banners, and interactive elements create noise. Clean markdown versions give AI systems direct access to the actual content.

### Which Pages to Prioritise

Create markdown versions of your 5 highest-value pages. These are typically:

1. Homepage
2. Main product or service page
3. About page
4. Most important sector/industry page
5. Pricing or contact page

### Serving Approach

**Option A: Static files**
Create `.md` files alongside your HTML pages. Simplest approach for static sites.

**Option B: Automated middleware (recommended for dynamic sites)**
Create middleware that:
1. Detects requests ending in `.md`
2. Fetches the corresponding page content
3. Strips HTML to markdown (use `turndown` for JS, `markdownify` for Python, `ReverseMarkdown` for .NET)
4. Returns with `Content-Type: text/markdown` and `X-Robots-Tag: noindex`

The `noindex` header is critical - it prevents Google treating markdown versions as duplicate content.

### Content Guidelines

- Extract content only - no navigation, footer, cookie banners, scripts
- Preserve heading hierarchy (H1, H2, H3)
- Include all text, tables, and lists
- Do not include images (LLMs cannot process images from markdown files)
- Add canonical reference at the top:

```markdown
<!-- Canonical: https://example.com/about/ -->
# About Us
...
```

## PART 5: META TAGS

Every page must have these in the `<head>`. Unique per page.

```html
<!-- Title - unique per page, under 60 characters -->
<title>[Page Title] | [Company Name]</title>

<!-- Description - unique per page, 150-160 characters -->
<meta name="description" content="[Concise description of what this page contains.]" />

<!-- Robots - explicitly allow indexing -->
<meta name="robots" content="index, follow" />

<!-- Canonical URL - must match the page URL exactly -->
<link rel="canonical" href="https://example.com/this-page/" />

<!-- Open Graph (critical for LinkedIn/social sharing) -->
<meta property="og:type" content="website" />
<meta property="og:title" content="[Page Title] | [Company Name]" />
<meta property="og:description" content="[Description - can match meta description]" />
<meta property="og:url" content="https://example.com/this-page/" />
<meta property="og:image" content="https://example.com/images/og-image.png" />
<meta property="og:site_name" content="[Company Name]" />
<meta property="og:locale" content="en_GB" />

<!-- Twitter/X card -->
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:title" content="[Page Title]" />
<meta name="twitter:description" content="[Description]" />
<meta name="twitter:image" content="https://example.com/images/og-image.png" />

<!-- AI discovery -->
<link rel="llms" href="https://example.com/llms.txt" />
```

### OG Image Requirements

- Dimensions: 1200x630px (LinkedIn, X, Slack, most platforms)
- Format: PNG or JPG
- Logo on branded background with company name/tagline
- Create page-specific OG images for key pages where possible

### Common Mistakes

- **Missing descriptions** - every page needs a unique one
- **Duplicate titles** - every page must have a distinct title
- **noindex left from development** - check every page, including in HTTP headers
- **Canonical pointing to localhost or staging** - must point to the live production URL
- **og:image using a relative path** - must be a full absolute URL

## PART 7: GEO CONTENT OPTIMISATION

These six techniques improve AI citation rates by 30-40% according to Princeton research.

### 7.1 Lead with Direct Answers

Every page section should open with a direct, factual statement before providing context. AI systems extract short passages, so the first sentence is most likely to be cited.

**Bad:** "In today's rapidly evolving technological landscape, organisations are increasingly recognising the importance of..."

**Good:** "[Product Name] scans configurations against 25 security rules and produces graded reports in under 5 seconds."

Review all pages. Rewrite opening paragraphs to lead with the fact, not the preamble.

### 7.2 Include Statistics and Data Points

AI engines heavily favour content with specific, verifiable numbers. Wherever a statistic exists, make it prominent.

Instead of: "Our tool is very fast"
Write: "Typical scan time: 1-5 seconds across 7 platforms"

Instead of: "We help organisations save money"
Write: "AI-accelerated development reduces delivery costs by 60-80% on suitable programmes"

### 7.3 Use Precise Technical Terminology

AI engines match user queries to content using specific terms. Use the exact phrases your audience would search for.

- Use the proper name of standards, frameworks, and certifications
- Use industry-specific terminology, not generic alternatives
- Include acronyms alongside full names on first use

### 7.4 Structure Content for Extraction

- Clear heading hierarchy: H1 > H2 > H3 > H4
- One topic per section
- Each section independently understandable without the rest of the page
- Short paragraphs (2-3 sentences maximum)
- Tables for comparative data (AI systems parse tables well)

### 7.5 Add "Last Updated" Timestamps

AI engines weigh recency when selecting sources. Add a visible "Last updated: [Month Year]" to every major page. Use a format like: `Last updated: June 2026`

Place at the bottom of the main content area, above the footer.

### 7.6 Build FAQ Content Aggressively

FAQ sections are gold for GEO. AI engines love question-answer pairs because they map directly to user queries.

**Add FAQ sections to every major page.** Aim for 4-6 questions per page. Write questions in the exact phrasing a user would ask an AI:

- "What is [your product]?"
- "How much does [your service] cost?"
- "Is [your product] free?"
- "Who built [your product]?"
- "What platforms does [your product] support?"
- "Does [your company] have [certification]?"

Wrap all FAQ sections in FAQPage JSON-LD schema (see Part 6).

### Template

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://example.com/</loc>

    <changefreq>monthly</changefreq>
```

| Page Type | Priority | Change Frequency |

| Blog/insights index | 0.7 | weekly |
| Individual articles | 0.6 | monthly |

## PART 10: SERVER-SIDE RENDERING CHECK

This is critical. If the site is a client-side rendered SPA (React, Vue, Angular without SSR/SSG), the initial HTML response is an empty shell with just JavaScript. Search engines and AI crawlers struggle to index these.

### Quick Test

```bash
curl -s https://example.com/ | grep -c '<h1\|<h2\|<p'
```

If the count is zero or very low, the site needs server-side rendering or static generation enabled.

### Manual Test

View source in the browser (Ctrl+U). If you see actual page headings, paragraphs, and content in the raw HTML, SSR/SSG is working. If you just see `<div id="root"></div>` and `<script>` tags, it is not.

### Framework Fixes

- **Next.js:** Use `getStaticProps` or `getServerSideProps` (Pages Router) or Server Components (App Router)
- **Nuxt:** SSR is enabled by default. Check `nuxt.config` for `ssr: false`
- **Gatsby:** Static generation by default. Should be fine.
- **Create React App:** No SSR. Migrate to Next.js or use a pre-rendering service like Prerender.io
- **Angular:** Use Angular Universal for SSR
- **Vue CLI:** Use Nuxt or add `vue-server-renderer`

## PART 11: SEARCH ENGINE REGISTRATION

These steps cannot be automated. Do them manually.

### Google Search Console

1. Go to https://search.google.com/search-console
2. Add property with your domain
3. Verify ownership (DNS TXT record recommended, or HTML file upload)
4. Submit sitemap: enter `sitemap.xml` in the Sitemaps section
5. Use URL Inspection to request indexing of your 5 most important pages
6. Check back after 48 hours for indexing status

### Bing Webmaster Tools

1. Go to https://www.bing.com/webmasters
2. Add site and verify
3. Submit sitemap
4. Use URL Submission for priority pages
5. Bing feeds results to DuckDuckGo, Yahoo, and several AI systems

### LinkedIn Post Inspector

1. Go to https://www.linkedin.com/post-inspector/
2. Enter your homepage URL
3. Verify OG tags render correctly (title, description, image)
4. If incorrect, fix OG tags and re-inspect. LinkedIn caches aggressively so you may need to wait or use the inspector to force a refresh
