## 1. What this website is

The Power In The Pause® is a premium lifestyle brand centred on emotional wellbeing, creativity and intentional living. This is not a coaching website. It should feel like a calm, spacious, quietly confident editorial brand — closer to Aesop, Kinfolk or a fine art gallery than to any coach's site.

People should feel themselves exhale as they move through the pages.

The site's core message: pausing is not doing nothing. Pausing is where clarity, creativity and transformation begin.

### Brand hierarchy (never violate this)

- **The Power In The Pause®** — the master brand. Everything sits beneath it. The homepage establishes it first, before any product.
- **The Art of Pausing®** — the creative practice and flagship iPhone app.
- **The Pause to Power Pathway®** — the 1:1 coaching programme.
- **Emotional Bioharmonising®** — the emotional wellbeing programme (4-week course).
- **The Spiral Shift™** — the free introduction (a 5–10 minute guided drawing experience).

Nothing competes with the master brand. Use ® / ™ marks on the first mention of each name per page, then drop them for readability.

### The visitor journey

Every page decision should serve this sequence:

1. Discover The Power In The Pause® — what it is, why it matters.
2. Understand why creativity belongs alongside sleep, movement and nutrition as an essential pillar of wellbeing.
3. Discover The Art of Pausing® — drawing as a vehicle for awareness. Not art lessons. Not therapy.
4. Read real stories.
5. Meet Katie — without making her the hero. The work is the hero.
6. Choose where to begin: The Spiral Shift™, The Art of Pausing®, Coaching, or Emotional Bioharmonising®.

The visitor should never wonder "what do I do next?"

## 2. Tech spec

### Framework and hosting

- **Next.js 14+ (App Router), TypeScript, Tailwind CSS.**
- The site is fully static — no server-side data, no database, no auth. Use `output: 'export'` in `next.config.js` so the build produces plain static files. This keeps the free Azure tier comfortable.
- **Hosting:** free Microsoft Azure Web App. Two acceptable setups — pick whichever deploys cleanly:
  1. **Azure Static Web Apps (Free plan)** — the natural fit for a static Next.js export. GitHub Actions workflow generated by Azure.
  2. **Azure App Service (F1 Free, Linux, Node 20)** — serve the exported `out/` directory (e.g. with `serve` or a minimal Express static server). Deploy via GitHub Actions using `azure/webapps-deploy`.
- Custom domain: thepowerinthepause.co.uk with HTTPS. Note the F1 App Service tier does not support custom domains — if App Service is chosen, the plan needs to be B1 or the site should go on Static Web Apps Free (which does support custom domains). Flag this decision to Katie before deploying.
- Set up `www.` → apex redirect (or the reverse — be consistent).

### Repo conventions

- Git from the first commit. Meaningful commit messages. `main` is deployable at all times.
- GitHub Actions workflow: build → export → deploy on push to `main`.
- Images in `/public/images/`, brand assets in `/public/brand/`.
- One component per file, `/components` directory. Keep the component count low — this is a simple site and should stay that way.

### The only interactive pieces

- Navigation dropdowns (Art, Shop) — accessible, keyboard-navigable, work on touch.
- Shopify Buy Button embeds on the two Shop pages (see §7).
- Contact form (see Contact page, §6.8).
- Email capture for The Spiral Shift™ (see §7).
- Subtle scroll-in animations (see Motion, §3).

Nothing else. No chat widgets, no popups, no exit-intent, no cookie-consent theatre beyond what's legally required (a single quiet banner if analytics are added).

## 3. Design system

### Feel

Elegant. Editorial. Calm. Thoughtful. Premium. Warm. Spacious.

Never: corporate, loud, busy, over-designed, "life coach", hustle culture, overly spiritual, overly clinical.

Design references: Apple, Aesop, Kinfolk Magazine, Calm app, a luxury boutique hotel, a fine art gallery. When unsure between two design options, always choose the calmer, simpler, more spacious, more timeless one. The website should feel like taking a deep breath.

### Colour palette

| Role | Colour | Hex | Usage |
|---|---|---|---|
| Background | Warm cream | `#FCFBF5` | Default page background (this is Katie's confirmed brand cream) |
| Alternate background | Soft white | `#FFFFFF` | Alternating sections, cards |
| Primary text / headings | Deep navy | `#1C2A44` | All body text and headings — never pure black |
| Accent | Metallic gold | `#C6A55C` | Fine rules, small details, hover states, script accents. A subtle gold — never shiny or gradient-heavy |
| CTA only | Magenta | `#D72C8F` | Katie's confirmed brand magenta. **Important call-to-action buttons only.** At most one magenta element visible per screen. Everything else stays quiet so the magenta feels intentional |

No other colours. Generous whitespace is part of the palette.

### Typography

- **Headings:** an elegant serif. Recommended: Cormorant Garamond (via `next/font/google`). Large, light weight, generous letter spacing on small caps labels.
- **Script (emphasis only):** a beautiful handwritten script for the occasional single word or short phrase — e.g. the word *pause* inside a heading. Recommended: Pinyon Script or Parisienne. Use it perhaps once per page. Never for whole sentences.
- **Body:** a clean sans serif. Recommended: Jost or Inter, comfortable size (18px base), line-height around 1.7, navy on cream.

Typography carries the emotion. Copy blocks stay narrow (max ~65ch). Headings get room to breathe above and below.

### Logo

Use the current logo from katiecoopercoaching.co.uk, same style and treatment. `[TODO: Katie to export the logo files (SVG preferred, plus PNG fallback) from the current site or from her designer and drop them into /public/brand/]`. Header: logo left, nav right. Footer: smaller logo, muted.

### Photography

Black and white only. Natural, real, quiet, documentary rather than posed. Never stock photography. `[TODO: Katie to supply photos — herself, hands drawing, the deck and journal, studio/artwork shots]`. Until real photos arrive, use neutral cream/navy placeholder blocks — not placeholder stock images.

Product photography (deck, journal, artwork) may be in colour where the product itself is the point; everything featuring people or place is black and white.

### Motion

- Soft fade-and-rise on scroll (opacity + ~12px translate, 600–800ms, ease-out). Once per element, no repeat on re-scroll.
- Slow, subtle. Nothing bounces, slides in from the side, or auto-plays.
- Respect `prefers-reduced-motion`: disable all animation.

## 4. Voice and copy rules

Simple. Warm. Confident. Written as though speaking quietly to one person.

- Short sentences. Short paragraphs. Lots of white space between them.
- Never salesy. No buzzwords, no hype, no fake urgency, no manipulation, no overpromising. No "transform your life in X days", no countdown timers, no "spots are filling fast".
- Less copy, better copy. Every sentence earns its place. When editing, cut before adding.
- Avoid coach-speak and spiritual jargon (no "soul-led", "alchemy", "vibration", "manifest"). Avoid clinical language too. Plain, human words.
- British English throughout (harmonise, colour, programme).

The copy in §6 below is the approved draft. Claude Code may tighten it but should not inflate it.

## 9. Accessibility and performance

- WCAG AA contrast (navy on cream passes; check gold text — use gold for decoration, not for body text).
- Full keyboard support for dropdowns and the mobile menu; visible focus states (thin gold outline).
- All images get meaningful `alt` text; decorative ones get `alt=""`.
- `next/image` (with `unoptimized: true` under static export, or pre-optimised assets), lazy loading below the fold, fonts via `next/font` with `display: swap`.
- Lighthouse targets: 95+ across the board. A calm site should also be a fast one.
