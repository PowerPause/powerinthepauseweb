# Lessons

Project-level lessons learned, feeding the master library's improvement loop.

**Rule (v1.6.1):** any fix that costs more than one session, or any pattern-shaped defect found in 3+ places, gets an entry here before the work is considered complete. Entry format: what went wrong, cost (time/tokens), root cause, the rule that would have prevented it.

At project retrospectives, the orchestrator reviews this directory and flags entries that should be promoted to the master library (template conventions, mandated tests, skills). The TriTech project's lessons document is the founding example: see `version-control/lessons/` in the master library.
