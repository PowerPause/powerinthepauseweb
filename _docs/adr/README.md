# Architecture Decision Records

One file per major architectural decision: `NNN-short-title.md`. Created at the architecture phase, when changing a major technology choice, or whenever a future developer would ask "why did they choose X?"

A TriTech-project lesson: decisions made ad-hoc without ADRs had to be reverse-engineered when problems arose. The orchestrator should propose an ADR whenever a Gate 0-5 decision, hosting choice, data store, auth approach, or framework selection is made.

Use `template.md` in this directory as the starting point.
